# Issue Template

Use one issue per idea. Keep the body short enough to read on a phone.

```markdown
Title: <short idea title>

## Capture
- Source: Telegram / note / URL / conversation / other
- Date: <YYYY-MM-DD>
- Original text:
  > <paste the raw capture here>

## What this is
<1-3 sentences in Tony's words, or the closest safe paraphrase>

## Why it matters
- <why this is worth keeping>

## Related ideas
- <link to related issue or repo>

## Next step
- <one concrete action>

## Suggested labels
- capture
- <domain label>
- <state label if needed>

## Promotion target
- <specialist skill or repo, if known>
```

If the input is a Telegram dump, split it into multiple issues when the items
are clearly separate. If the split is uncertain, keep the candidate items in
one issue and mark the ambiguity.
